# 🐳 Honeygain Автономный Docker Node

Отказоустойчивый и полностью автоматизированный стек для развертывания ноды **Honeygain** с интеграцией **Watchtower** для автообновлений и встроенным режимом **JumpTask (JMPT)**.

## 📌 Для чего нужен этот проект?

**Honeygain** — это краудсорсинговая сеть, которая позволяет монетизировать неиспользуемую пропускную способность вашего интернет-канала. Компании используют этот трафик для маркетинговых исследований, сравнения цен и верификации рекламы. 

### Почему именно Docker-контейнер, а не приложение для Windows/Android?
* **Максимальная выживаемость**: Контейнер настроен на политику `restart: always`. Если упадет интернет, перезагрузится сервер или зависнет служба, Docker мгновенно поднимет ноду заново.
* **Экономия ресурсов**: Контейнер работает без графического интерфейса и потребляет всего **~10-15 МБ** оперативной памяти и около **0.05%** процессора.
* **Автономия 24/7**: Встроенный инструмент Watchtower раз в сутки проверяет наличие обновлений. Нода всегда работает на самой свежей и прибыльной версии без вашего участия.
* **Безопасность**: Лимиты на логи (`max-size: 10m`) защищают ваш жесткий диск от переполнения в случае сбоев.

## 🛠️ Быстрый запуск

### 1. Подготовка конфигурации
Склонируйте репозиторий, создайте в корневой директории файл `.env` (он находится в `.gitignore` и защищен от отправки в публичный доступ) и заполнить своими данными:

```env
HONEYGAIN_EMAIL=your_email@gmail.com
HONEYGAIN_PASSWORD=your_honeygain_secure_password
HONEYGAIN_DEVICE=docker_node_1
```
*Важно: Если ваш аккаунт зарегистрирован через Google Auth, предварительно сделайте сброс пароля на сайте Honeygain, чтобы задать текстовый пароль для CLI-авторизации.*

### 2. Запуск стека
Запустите проект одной командой:
```bash
docker compose up -d
```

## 🎛️ Управление проектом

* **Просмотр статуса контейнеров**: `docker compose ps`
* **Просмотр логов ноды**: `docker logs honeygain_app`
* **Живой мониторинг трафика и ресурсов**: `docker stats honeygain_app`
* **Остановка ноды**: `docker compose down`

## ⚠️ Правила использования сети
Honeygain строго запрещает запуск более **1 устройства на один IP-адрес**. Убедитесь, что этот контейнер не делит один IP с запущенным приложением на Windows или Android, иначе вы получите статус `Network Overused`. Проект работает только на **Residential (домашних)** или **Mobile** IP-адресах (IP дата-центров/VPS блокируются системой).
